# NKCO 音乐会内容添加规范

## 目标
- 网页“近期音乐会”只展示指定推文链接对应的音乐会内容。
- 保留并持续展示两场基础音乐会：
  - 夏至天开（`https://mp.weixin.qq.com/s/lsWojZHqV-6LS1XUGPB9aQ`）
  - 三周年音乐会（`https://mp.weixin.qq.com/s/z7CpjcfgzncmD0q-wDyJBw`）

## 数据来源
- 所有输入链接维护在 `data/concert_links.json`。
- 更新命令：
  - `python3 scripts/update_concerts.py`
- 输出文件：
  - `data/concerts.json`
- 海报下载目录：
  - `assets/images/concerts/`

## 抽取规则
- 每场音乐会优先定位“标题包含 `预告`”的推文。
- 同一场音乐会有多条时，使用该场“第一条（最早发布）的预告推文”作为展示来源。
- 展示字段最少包含：
  - 音乐会名称
  - 时间
  - 地点
  - 海报
  - 源推文链接

## 时间与地点格式
- 时间统一为中文日期格式：`YYYY年M月D日`（页面不展示具体时分）。
- 地点统一使用精简场馆格式（去除节目单和说明性文本）：
  - 线下场地示例：`南开大学八里台校区田家炳音乐厅`
  - 线上场地示例：`线上直播（Bilibili）`
- 无法可靠提取地点时，统一回退：`详见公众号推文`

## 海报规则
- 海报来源应尽量贴近公众号封面图，优先使用推文 `msg_cdn_url`。
- 页面端统一使用横屏展示（16:9 区域），允许裁切以保证统一布局。

## 页面规则
- “近期音乐会”每行 3 场（桌面端）。
- 移动端可自动降为 2 列或 1 列。
- 排序按音乐会时间降序（越新越靠上）。
- 未提取到完整信息时：
  - 时间回退为推文发布时间
  - 地点回退为“详见公众号推文”

## 变更流程
1. 在 `data/concert_links.json` 添加或更新推文链接。
2. 运行 `python3 scripts/update_concerts.py`。
3. 本地预览并核对：
   - `python3 -m http.server 8000`
   - 打开 `http://localhost:8000`
4. 确认“近期音乐会”满足 3 列横版、时间倒序、链接可跳转。
